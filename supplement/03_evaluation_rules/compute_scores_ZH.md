# compute_scores_ZH.md

## 文件定位与角色

本文件是 `compute_scores.py` 的**中文说明性版本**，用于解释评分与汇总逻辑的设计意图与安全边界。

- **本文件不作为可执行脚本**
- **不作为评测或统计的入口**
- **不参与任何结果生成流程**

本 artifact 中所有用于论文与补充材料的定量结果，均已**预先计算并固定存储**于：

```
supplement/04_results/
```

评审无需、也不应运行任何评分脚本即可完成审计。

---

## 与 `compute_scores.py` 的关系

- `compute_scores.py`：
  - 唯一的评分与聚合逻辑实现
  - 用于生成已固定的结果文件

- `compute_scores_ZH.md`（本文件）：
  - 对上述逻辑的**中文解释与审计说明**
  - 不包含任何权威计算结果
  - 不引入任何新的评分路径

在出现任何歧义时，应以 **`compute_scores.py` 的实现逻辑**为准，而非本说明文档。

---

## 设计原则

本评分逻辑的设计遵循以下原则：

1. **分析级可复现**  
   本 artifact 仅承诺：
   - 固定输出 → 评分规则 → 汇总结果 的逻辑一致性

2. **避免 API 依赖与非确定性执行**  
   不要求评审复现模型推理或在线接口调用。

3. **可审计而非可执行**  
   所有脚本均服务于逻辑透明与审计，而非复现实验运行。

---

## 评分流程概览

从逻辑上，评分与汇总流程可抽象为以下步骤：

1. **输入**  
   - 单次实验运行目录（包含固定模型输出）
   - 对应的评分 rubric / 规则定义

2. **校验**  
   - 目录结构存在性检查
   - 评分规则文件完整性检查

3. **评分与汇总（概念层面）**  
   - 根据 rubric 对固定输出进行逐项判定
   - 汇总得到结构化的评分结果

4. **输出**  
   - 生成结构化的 summary 结果文件

本说明文件不描述具体实现细节，以避免形成“第二实现路径”。
