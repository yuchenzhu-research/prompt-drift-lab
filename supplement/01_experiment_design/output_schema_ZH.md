# 标准输出结构

本文件定义 Prompt Drift Lab 中“结构化输出任务”的**标准输出结构**。
评测关注：模型是否**按要求完成结构化任务**，而非回答内容的正确性。

该 schema 是 Rubric 的上游真值源：
- A_structure：是否按规定结构输出
- B_snapshot_constraint：事实快照是否“只陈述不分析”
- C_actionability：搜索指令是否可执行
- D_completeness：是否完整覆盖要求部分
- E_drift_failure：是否发生明显漂移

---

## 1. 输出必须满足的整体约束

被测模型的最终输出**必须**满足以下所有约束：

1. **仅包含 3 个一级部分**，且顺序固定；标题需与 raw PDF 中的编号与方括号样式保持一致：

   **第 1 部分**

   ```
   1. [事实快照]
   ```

   **第 2 部分**

   ```
   2. [ChatGPT 联网搜索指令]
   ```

   **第 3 部分**

   ```
   3. [模型思维指令]
   ```

   > 示例（raw PDF）：`3. [Gemini 深度挖掘指令]`

2. **不得添加任何额外标题、前言或结语**

   - 不得出现“好的 / 当然 / 以下是 / 希望对你有帮助”等寒暄性文本
   - 不得添加“附录 / 补充 / 备注 / 注意事项 / FAQ”等额外结构

3. **不得直接回答原始问题 Q 的内容本身**。本任务的角色是“提示词生成器 / 研究助理”，输出应仅包含：

   - 一个事实快照
   - 两段可直接执行的检索 / 研究指令

4. **语言**：中文

---

## 2. 结构模板

输出必须严格按以下模板生成：

## 1. [事实快照]
在此处用 ≤50 字，冷静客观地陈述“现象/结论的表象”。不得解释原因，不得给建议，不得下定义。

## 2. [ChatGPT 联网搜索指令]
在此处给出一段可直接复制给 ChatGPT 的“联网搜索 + 研究整理”指令。必须可执行、含来源要求与交付物要求。

## 3. [模型思维指令]
在此处给出一段可直接复制给模型的“深度研究 / 广搜 / 对比分析”提示词。
