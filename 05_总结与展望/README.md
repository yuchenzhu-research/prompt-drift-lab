# 总结与展望（Summary & Outlook）

> 本文档用于 **“总结实验现象 + 连接外部研究 + 提出下一步可复现路线”**。
>
> **重要说明（修正）**：本项目当前版本 **未实现任何自动化扰动生成器（perturb）**。以下内容已严格对齐你**实际完成并落盘的实验资产**，不再包含任何未实施的方法或工具建议。

---

## 1. 我到底做出了什么

本项目围绕 **Prompt Drift**（提示词微小变化导致的遵循率下降、格式崩坏、语义漂移与约束漏失）做了一个可复现的最小实验闭环：

- **输入端**：在固定任务与题集条件下，人工构造少量、可解释的提示词差分版本，用于暴露指令遵循与结构稳定性问题。
- **输出端**：保留不同模型在不同提示词条件下的原始输出（PDF），并形成结构化结果汇总（CSV）。
- **评测端**：采用跨模型互评 + 模型自评作为补充对照，配套明确的评测协议与评委提示词（见 `03_评测规则/`）。

> 重要边界：本项目的所有 drift 现象，**均来自人工设计的提示词版本差异**，不存在自动扰动、随机变换或大规模 prompt 扫描。

---

## 2. 主实验范围与取舍说明

为了避免读者误解，本仓库明确区分：

- **主实验（可复现）**：围绕 `Prompt B` 及其三个手工构造的变体（long / weak / conflict）进行对照评测与统计汇总。
- **先导探索（pilot）**：`Prompt A` 仅用于早期探索与问题暴露，不参与主实验的定量比较。

**方法取舍说明**：选择 `Prompt B` 作为主实验基线，并非随意决策，而是基于先导探索中 A/B 在可执行性、格式稳定性与工程可复核性上的对照结果；相关证据与对比记录已在 `06_` 中集中说明。

---

## 3. 关键观察（基于已完成实验）

结合 `summary.csv` 与各类分组统计结果，本项目能够支持以下**不超出证据范围**的观察：

1. **即使是少量、人工设计的提示词差分，也足以触发显著的指令遵循与格式稳定性变化**。
2. **更长或更详细的提示词并不保证更高的稳定性**；在部分条件下反而引入约束稀释或执行回退。
3. **弱化约束表述（如将 must 改为 should）会显著提高 drift 发生概率**。
4. **存在指令张力（conflict）的版本，更容易暴露模型的隐式优先级选择行为**。

---

## 4. 外部研究的作用（解释视角，而非方法补充）

本节仅用于将已观察到的现象，与现有研究中的概念进行对齐，而**不意味着本项目已实现或复现相关方法**：

- Prompt brittleness / prompt sensitivity
- instruction following stability
- format robustness

这些研究帮助解释：为何即便在极小输入改动下，LLM 的输出行为仍可能出现明显漂移。

---

## 5. 方法局限与未来工作（严格区分）

### 5.1 当前版本的明确局限

- 提示词差分完全依赖人工设计，覆盖面有限。
- 未引入自动扰动、随机化或系统性扫描机制。
- 结论仅适用于当前提示词空间与评测设置。

### 5.2 未来可能的扩展方向（未实施）

> 以下内容仅作为**可能的后续研究方向提示**，不构成本项目当前版本的方法组成部分：

- 若未来引入自动化扰动生成或更大规模 prompt 扫描，应单独作为新版本或新实验报告，不与当前结果混用。

---

## 6. 项目定位总结

本项目的价值在于：

- **把 prompt 变化当作实验变量，而不是技巧调参**。
- **用少量、可解释的对照实验，明确展示 drift 的存在与类型**。
- **通过严格的文件结构与协议，使结论可复核、可复现**。

它不是一个“大而全”的 benchmark，而是一个 **边界清晰、证据诚实的最小研究工程**。
