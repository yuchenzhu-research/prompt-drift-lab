# 04_实验结果（Results Index）

> 目标：读者 **30 秒内**定位：`summary.csv` / `main_method_*` / `supporting_method_*` / 有效&无效评测（valid & invalid）。

## 1) 30 秒入口（从这里开始）

1. 打开 **`summary.csv`**：一行 = 一次 run 的汇总（推荐先看它）。
2. 看 **主方法（main method）**：`main_method_*`（跨模型互评的主要结论与证据）。
3. 看 **辅助方法（supporting method）**：`supporting_method_*`（自评/补充对照，默认不作为主结论）。
4. 若发现缺项/不可比，去看 **有效/无效评测**（valid & invalid）的分桶与 flags。

> 注：本目录只做“结果与分析入口”。评测规则与 judge 提示词在 `03_评测规则/`。

---

## 2) 结果文件一览（按读者常见问题组织）

### A. 我想快速知道“最终数字/结论在哪里？”
- **`summary.csv`**：总表入口（推荐用于复核与复现对齐）。
- **`03_实验结果分析.md`**：对结果的解释、失败模式与漂移归因线索。

### B. 我想看“主方法”产物（main method）
在本目录中搜索/定位以下前缀：
- `main_method_*`

通常包含（以实际文件为准）：
- `main_method_*.csv`：逐题逐维度评分 / 汇总统计
- `main_method_*/`：对应的 judged 输出、证据片段、或可复核中间产物

### C. 我想看“辅助方法”产物（supporting method）
在本目录中搜索/定位以下前缀：
- `supporting_method_*`

说明：辅助方法默认只用于 **sanity check/机制线索**，不混入主结论（除非你在统计口径里显式声明）。

### D. 有效评测 vs 无效评测在哪里？（valid & invalid）
- **有效评测（valid）**：请在仓库中定位 `有效评测` 相关目录（通常在 `02_跨模型评测结果/有效评测/`）。
- **无效评测（invalid）**：请在仓库中定位 `无效评测` 相关目录（通常在 `02_跨模型评测结果/无效评测/`）。
  - 无效评测文件应包含可机器读取的 `flags`（如 `UNPARSABLE_OUTPUT` / `PROTOCOL_VIOLATION` 等），用于解释“为何不可比/不可纳入统计”。

> 如果你只在 04 目录阅读：建议先看 `03_实验结果分析.md` 中“Invalid 与 Failure Modes”一节，再回到 `02_跨模型评测结果/无效评测/` 查样本。

## 3) 读者复核路线（不依赖脚本）

- **路线 1：只看结论**
  - `summary.csv` → `03_实验结果分析.md`

- **路线 2：结论 + 证据链（推荐）**
  - `summary.csv` → `main_method_*`（找到对应 judged 输出/证据）→ 回到 `03_实验结果分析.md` 对照失败模式标签

- **路线 3：排查异常/缺项**
  - `summary.csv` 发现缺项 → 去 `无效评测（invalid）` 查对应 `flags` → 回到 `03_实验结果分析.md` 查看该类 failure 的解释与处理策略

## 4) 常见读者问题（FAQ）

### Q1：为什么有 `main_method_*` 和 `supporting_method_*` 两套？
A：主方法用于形成可比的主结论；辅助方法用于机制线索或一致性检查，默认不混入主统计。

### Q2：为什么有些结果被标为 invalid？
A：invalid 表示“不可作为定量依据”，常见原因是协议未执行、输出不可解析、覆盖不完整或上下文错位等；具体标签与示例见 `03_实验结果分析.md`。
