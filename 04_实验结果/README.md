# 04｜实验结果（v2）

本目录汇总 Prompt Drift 实验的**原始证据、结构化结果、分析与讨论**。  
阅读顺序按“结果 → 分析 → 归因 → 讨论”组织，尽量贴近 workshop/论文结果章节。

---

## 快速导航（建议从这里开始）

- **只看结论与可行动建议**：`04_讨论与设计启示.md`
- **想先看到“发生了什么”**：`01_结构遵循结果总览.md`
- **想知道“漂移集中在哪里”**：`02_结构遵循与指令漂移分析.md`
- **想看失败模式 taxonomy（可复用）**：`03_失败模式归因分析.md`

---

## 文件说明（Results → Analysis → Attribution → Discussion）

- `01_结构遵循结果总览.md`  
  **Results**：按规则/题目汇总硬性合规表现（发生了什么）。

- `02_结构遵循与指令漂移分析.md`  
  **Analysis**：从规则层/段落层/题型层定位漂移集中区（漂在哪里）。

- `03_失败模式归因分析.md`  
  **Attribution**：将现象收敛为失败模式 taxonomy，并给出触发条件假设（可检验）。

- `04_讨论与设计启示.md`  
  **Discussion**：把失败模式翻译成设计原则，并输出最小改动清单（可直接粘贴到提示词）。

---

## 证据与数据（可复现性）

- `raw_outputs/`  
  原始输出证据（不清洗、不改写），用于抽查与复核。  
  **建议命名约定**：`{Prompt版本}_{Qid}_{trial}.md/pdf`  
  示例：`P1_Q3_t2.md`

- `data/`  
  结构化整理后的结果表/统计汇总（用于生成 01/02 中的表格）。

> 注：本项目关注“结构遵循/指令漂移”，不评事实正确性；评测规则见 `03_评测规则/`。

---

## 建议阅读路径

1) 先看 `01` 抓住总体合规表现  
2) 再看 `02` 明确漂移集中点（规则/段落/题型）  
3) 用 `03` 把现象固化为 failure taxonomy  
4) 最后用 `04` 产出“最小改动清单”
